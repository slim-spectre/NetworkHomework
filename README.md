## Завдання 3: Експерименти з ping та traceroute

### Відповіді на контрольні запитання:

*   **Скільки хопів до google.com?**
    Пакет пройшов через 12 проміжних пристроїв (маршрутизаторів).

*   **Який середній час відгуку (RTT)?**
    Середній час для кінцевого сервера склав **34 ms**. Найменшу затримку серед зовнішніх ресурсів показав сервер `1.1.1.1` (12 ms).

*   **Чи всі проміжні маршрутизатори відповідають?**
    У моєму cmd відповіли всі 12 хопів. Проте, деякі вузли можуть не відповідати (відображатися як `* * *`), якщо на них налаштовано ігнорування ICMP-пакетів з міркувань безпеки або через низький пріоритет обробки таких запитів під час високого навантаження.

*   **Яка різниця між ping до локального шлюзу та публічного сервера?**
    1. **Затримка (Latency):** До локального шлюзу вона мінімальна (<1 ms), до публічних серверів — значно вища через фізичну відстань та кількість проміжних вузлів.
    2. **Параметр TTL:** При пінгуванні локального пристрою TTL зазвичай максимальний (наприклад, 64), тоді як для віддалених серверів він менший, оскільки кожен пройдений маршрутизатор зменшує це значення на одиницю.
    3. **Доступність:** Спроба ping адреси `192.168.1.1` завершилася невдачею, оскільки цей IP не належить до поточної підмережі хоста (`192.168.0.0/24`).
